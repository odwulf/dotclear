Mise à jour de plugins

Il est possible dans Dotclear 2 de spécifier dans sa configuration (PATH) plusieurs chemins de dossiers contenant des plugins. Or, les mises à jour des plugins se font dans le dernier répertoire du PATH. Il se peut, pour des raisons variées, que dans un des premiers, il existe un plugin avec une version inférieure à celle présente sur DotAddict. Dans ce cas préçis, vous avez deux choix possibles :
  • Autoriser la multi-installation : la nouvelle version du plugin sera téléchargé puis installée dans le dernier répertoire du PATH et l’ancien plugin recevera un fichier _disabled afin de le désactiver. L’ancienne version ne sera pas supprimée
  • Ne pas autoriser la multi-installation : la nouvelle version ira écraser l’ancienne dans le bon répertoire.
Par défaut, la multi-installation est désactivée mais vous pouvez changer ce comportement en modifiant la variable dainstaller_allow_multi_install dans about:config
