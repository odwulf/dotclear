It is possible, in Dotclear 2, to specify in the configuration (PATH) many paths of folders containing plugins. But, the plugin update are done only in the last folder specify in PATH. You may have, for several reasons, in other folders, an older plugin than currently available on DotAddict. In this case, you have two choices:
  • Autorize multi-installation: the newer version of the plugin will be downloaded and installed in the last folder of PATH and the older one will be disabled (a _disabled file will be created in his folder). This old version will not be deleted
  • Not autorize multi-installation : the newer version will overwrite the older one.
By default, multi-installation is disabled but you can change this behaviour by modifying the dainstaller_allow_multi_install variable in about:config
